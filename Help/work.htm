Боковая панель

Вкладка "Параметры задачи" позволяет менять физические и геометрические параметры задачи. Отдельно следует остановиться на задании краевых условий.
Параметр "Краевые условия для температуры" позволяет выбрать один из двух предопределенных наборов краевых условий, один из которых соответствует задачи бокового подогрева, а другой - задаче донного подогрева.
При подогреве сбоку температура  на левой и правой стенках считается постоянной. На левой стенке она равна 0, а на правой - 1. Верхняя и нижняя стенки теплоизолированы.
При подогреве снизу температура температура считается постоянной на верхней и нижней стенках (1 на нижней стенке и 0 - на верхней), а вдоль боковых стенок она меняется линейно от 0 до 1.
Набор параметров "Жесткие стенки" определяет краевые условия для вихря. Если стенка НЕ является жесткой - это означает, что на ней отсутствует трение и, сл-но, вихрь на такой стенке равен 0. В противном случае, трение на стенке есть и вихрь определяется в соответствии со следующей формулой:

Система уравнений Навье-Стокса, энергии и неразрывности является нелинейной. Поэтому для нахождения решения этой системы применяется метод простой итерации . Решение считается найденным, если значение невязки по нелинейности стало меньше чем 1e-6. Максимальное число итераций для данного метода регулируется соответствующим параметром. По достижении указанного числа итераций процесс решения прекращается в независимости от того, было ли найдено решение.
Вкладка "Управление отрисовкой результата" позволяет указать какую из пяти найденных функций нужно нарисовать.
Сохранить получившийся рисунок в файл можно с помощью соответствующей кнопки. Графический файл (в формате PNG) будет создан в папке ".\result". Имя фала будет соответствовать его содержимому: T.png - температура, Psi.png - функция тока, Omega.png - вихрь, Vx.png - X-компонента скорости, Vy.png - Y-компонента скорости.
Кнопка "Сохранить изображение в файл" доступна только если в качестве графического движка используется OpenGL движок. Если используется Surfer движок, то в данной кнопке нет необходимости, поскольку Surfer при обработке данных сам создает требуемые графические файлы (в формате EMF) и помещает их в папку ".\results".

 

Область взаимодействия с решателем

Решение задачи начинается после нажатия соответствующей кнопки. Справа от кнопки отображается текстовая информация о ходе решения, а именно: номер текущей итерации по нелинейности (т.е. номер текущей итерации метода простых итераций), максимальное число итераций по нелинейности (по достижении данного числа итераций решатель прекращает работу в независимости от того было ли найдено решение) и текущая невязка по нелинейности (решение считается найденным если значение невязки стало меньше чем 1e-6). Под текстовой информацией о ходе решения расположен ProgressBar, показывающий какое число итераций по нелинейности (в процентах от максимального числа итераций по нелинейности) уже завершено. Во время процесса решения доступ к настройкам программы блокируется. Также блокируется и кнопка запуска решателя. Разблокировка происходит после окончания работы решателя. Процесс вычисления можно в любой момент прервать нажатием соответствующей кнопки.
